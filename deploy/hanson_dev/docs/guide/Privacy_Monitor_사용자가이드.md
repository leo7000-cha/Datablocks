## 1. 개요

Privacy Monitor는 **개인정보보호법 제29조**에 따라 개인정보처리시스템의 접속기록을 수집, 저장, 조회, 분석하는 모듈입니다.

### 주요 기능

| 기능 | 설명 |
|------|------|
| **접속기록 수집** | 대상 DB의 Audit Log를 자동으로 수집 (5W1H 기반) |
| **해시 체인 무결성** | SHA-256 해시 체인으로 접속기록 위/변조 방지 |
| **이상행위 탐지** | 10가지 기본 규칙 기반 이상행위 자동 감지 |
| **접속기록 조회/다운로드** | 다양한 필터 조건으로 검색, Excel 다운로드 (감사 기록 포함) |
| **대시보드** | 실시간 현황 모니터링, 차트, 알림 요약 |

### 5W1H 수집 항목

| 항목 | 설명 | 예시 |
|------|------|------|
| **Who** | 접속자 | admin, user01 |
| **When** | 접속 일시 | 2026-04-06 14:30:00 |
| **Where** | 접속 IP | 192.168.1.100 |
| **What** | 작업 유형 | SELECT, UPDATE, DELETE |
| **Which** | 대상 DB/테이블 | COTDL.TBL_MEMBER |
| **How** | 접속 채널 | WEB, API, BATCH |

## 4. 대시보드

접속기록관리의 메인 화면으로, 전체 시스템 현황을 한눈에 파악할 수 있습니다.

### 4.1 요약 카드 (상단)

4개의 카드가 당일 기준 주요 지표를 표시합니다:

| 카드 | 내용 |
|------|------|
| **오늘 접속 건수** | 당일 수집된 접속기록 총 건수 |
| **이상행위 탐지** | 당일 탐지된 이상행위 알림 수 |
| **미처리 알림** | 아직 처리되지 않은 알림 수 (NEW + ACKNOWLEDGED) |
| **모니터링 시스템** | 활성 수집 대상 수 / 전체 수집 대상 수 |

### 4.2 차트

#### 시간대별 접속 추이 (꺾은선 그래프)
- X축: 0시 ~ 23시
- Y축: 시간대별 접속 건수
- 업무시간 외 접속 패턴을 파악하는 데 활용

#### 작업유형별 분포 (도넛 그래프)
- SELECT, UPDATE, DELETE, INSERT, DOWNLOAD 등의 비율
- DELETE, DOWNLOAD 비율이 높으면 점검 필요

### 4.3 최근 이상행위 알림

최근 5건의 이상행위 알림을 표시합니다.

| 열 | 설명 |
|----|------|
| 심각도 | HIGH(빨강), MEDIUM(노랑), LOW(초록), INFO(파랑) |
| 알림 | 이상행위 요약 설명 |
| 대상자 | 해당 사용자 |
| 탐지시간 | 탐지된 일시 |
| 상태 | 신규/확인/처리완료/무시 |

> "전체 보기" 링크를 클릭하면 **이상행위 알림** 페이지로 이동합니다.

### 4.4 시스템별 수집 현황

등록된 수집 대상 시스템의 상태를 표시합니다.

| 열 | 설명 |
|----|------|
| 시스템 | 수집 대상 시스템명 |
| DB유형 | Oracle, MariaDB, MySQL 등 |
| 상태 | RUNNING(수집중), STOPPED(중지), ERROR(오류) |
| 최근 수집 | 마지막 수집 시각 |
| 누적 건수 | 총 수집된 접속기록 건수 |

## 5. 접속기록 조회

수집된 접속기록을 다양한 조건으로 검색하고, 상세 정보를 확인할 수 있습니다.

### 5.1 검색 필터

| 필터 | 설명 | 입력 방법 |
|------|------|-----------|
| **시작일** | 조회 시작 날짜 | 날짜 선택 (달력) |
| **종료일** | 조회 종료 날짜 | 날짜 선택 (달력) |
| **사용자** | 접속자 계정 | 텍스트 입력 (부분 일치) |
| **작업유형** | 작업 종류 | 드롭다운: SELECT / UPDATE / DELETE / INSERT / DOWNLOAD |
| **PII등급** | 개인정보 등급 | 드롭다운: 1급 / 2급 / 3급 |
| **테이블명** | 대상 테이블 | 텍스트 입력 (부분 일치) |

> 필터를 설정한 후 **[조회]** 버튼을 클릭합니다.

### 5.2 조회 결과 테이블

| 열 | 설명 |
|----|------|
| No | 순번 |
| 접속일시 | 접속 시각 |
| 사용자 | 접속자 계정 |
| 접속IP | 클라이언트 IP 주소 |
| 작업유형 | SELECT/UPDATE/DELETE/INSERT/DOWNLOAD (색상 구분) |
| 대상DB | 접근한 데이터베이스 |
| 대상테이블 | 접근한 테이블 |
| PII등급 | 1급(빨강)/2급(노랑)/3급(초록) |
| 결과 | SUCCESS(성공)/DENIED(거부) |
| 채널 | WEB/API/BATCH |

#### 작업유형 색상 기준

| 유형 | 색상 | 의미 |
|------|------|------|
| DELETE | 빨강 | 삭제 작업 (주의 필요) |
| DOWNLOAD | 노랑 | 다운로드 작업 (개인정보 유출 주의) |
| 기타 | 파랑 | SELECT, UPDATE, INSERT 등 |

### 5.3 상세 정보 보기

테이블의 행을 클릭하면 **상세 정보 모달**이 표시됩니다.

표시 항목:
- 접속일시, 사용자, 접속IP
- 작업유형, 대상 DB/테이블
- PII등급, 결과코드
- SQL문 (수집된 경우)
- **해시값** (무결성 검증용)

### 5.4 Excel 다운로드

1. 검색 조건을 설정하고 **[Excel]** 버튼을 클릭합니다.
2. **다운로드 사유 입력** 팝업이 표시됩니다.
3. 사유를 입력하고 확인을 누르면 Excel 파일이 다운로드됩니다.

> **감사 기록**: 모든 다운로드는 자동으로 감사 기록이 남습니다 (다운로드 일시, 사용자, 검색조건, 건수, 사유).

## 6. 이상행위 알림

이상행위 탐지 규칙에 의해 자동으로 감지된 알림을 관리합니다.

### 6.1 기본 탐지 규칙 (10가지)

| 코드 | 규칙명 | 심각도 | 설명 |
|------|--------|--------|------|
| R01 | 대량 조회 탐지 | HIGH | 1시간 내 1,000건 이상 조회 |
| R02 | 대량 다운로드 탐지 | HIGH | 1시간 내 10회 이상 다운로드 |
| R03 | 업무시간 외 접속 | MEDIUM | 22:00~06:00 접속 |
| R04 | 휴일 접속 탐지 | MEDIUM | 주말/공휴일 접속 |
| R05 | 접근 거부 탐지 | HIGH | 1시간 내 5회 이상 접근 거부 |
| R06 | 고등급 개인정보 접근 | MEDIUM | 1급 개인정보 접근 |
| R07 | 반복 조회 탐지 | LOW | 30분 내 동일 테이블 50회 이상 조회 |
| R08 | 민감정보 다운로드 | HIGH | 1급/2급 개인정보 다운로드 |
| R09 | 미등록 IP 접속 | MEDIUM | 등록되지 않은 IP에서 접속 |
| R10 | 장기 미사용 계정 접속 | LOW | 90일 이상 미접속 계정의 재접속 |

### 6.2 검색 필터

| 필터 | 설명 |
|------|------|
| **심각도** | HIGH / MEDIUM / LOW / INFO |
| **상태** | 신규 / 확인 / 처리완료 / 무시 |

### 6.3 알림 목록

| 열 | 설명 |
|----|------|
| 심각도 | 등급별 색상 배지 |
| 규칙 | 규칙 코드 (R01~R10) |
| 알림 내용 | 이상행위 요약 |
| 대상자 | 해당 사용자 |
| 탐지시간 | 탐지 일시 |
| 상태 | 현재 처리 상태 |
| 처리 | [처리] 버튼 (미처리 건만 표시) |

### 6.4 알림 처리 방법

1. 처리할 알림의 **[처리]** 버튼을 클릭합니다.
2. **처리 의견**을 입력하는 팝업이 표시됩니다.
3. 의견을 입력하고 확인을 누르면 상태가 **"처리완료"**로 변경됩니다.

#### 알림 상태 흐름

```
신규(NEW) → 확인(ACKNOWLEDGED) → 처리완료(RESOLVED)
                                → 무시(DISMISSED)
```

## 7. 수집 대상 관리

접속기록을 수집할 대상 시스템을 등록하고 관리합니다.

### 7.1 수집 대상 등록

1. **[수집 대상 등록]** 버튼을 클릭합니다.
2. 등록 모달에서 아래 정보를 입력합니다:

| 항목 | 필수 | 설명 |
|------|------|------|
| **시스템명** | O | 수집 대상 시스템의 이름 (예: "운영DB", "정보계") |
| **수집방식** | O | DB_AUDIT / GENERAL_LOG / DLM_SELF |
| **DB유형** | O | Oracle / MariaDB / MySQL / MSSQL / Tibero |
| **연계 DB명** | - | DLM에 등록된 연계 DB 이름 |
| **호스트** | O | DB 서버 호스트명 또는 IP |
| **포트** | O | DB 서버 포트 (예: 3306, 1521) |
| **수집주기** | O | 분 단위 (기본값: 5분) |
| **설명** | - | 비고사항 |

3. **[저장]** 버튼을 클릭합니다.

#### 수집방식 안내

| 방식 | 설명 | 대상 DB |
|------|------|---------|
| **DB_AUDIT** | DB의 Audit Trail 조회 (Oracle Unified Audit 등) | Oracle |
| **GENERAL_LOG** | DB의 General Log 조회 | MariaDB, MySQL |
| **DLM_SELF** | DLM 자체 AOP로 자동 수집 | DLM 내부 작업 |

### 7.2 수집 제어

수집 대상 목록에서 각 시스템별로 아래 작업을 수행할 수 있습니다:

| 버튼 | 동작 |
|------|------|
| **재생(▶)** | 수집 시작 - 대상 DB의 접속기록을 수집 |
| **정지(■)** | 수집 중지 |
| **편집(✏)** | 수집 대상 정보 수정 |
| **삭제(🗑)** | 수집 대상 삭제 (확인 팝업) |

### 7.3 수집 상태

| 상태 | 색상 | 설명 |
|------|------|------|
| **RUNNING** | 파랑 | 수집 진행 중 |
| **STOPPED** | 회색 | 수집 중지됨 |
| **ERROR** | 빨강 | 수집 오류 발생 (연결 실패 등) |

## 8. 환경설정

시스템 설정, 탐지 규칙, 해시 무결성 검증을 관리합니다.

### 8.1 일반 설정

| 설정 키 | 기본값 | 설명 |
|---------|--------|------|
| LOG_RETENTION_DAYS | 365 | 접속기록 보관 기간 (일) |
| COLLECT_INTERVAL_MIN | 5 | 기본 수집 주기 (분) |
| HASH_ALGORITHM | SHA-256 | 해시 체인 알고리즘 |
| ALERT_ENABLED | Y | 이상행위 알림 활성화 여부 |
| BATCH_INSERT_SIZE | 1000 | 배치 입력 크기 |
| DOWNLOAD_REASON_REQUIRED | Y | 다운로드 시 사유 입력 필수 여부 |
| MAX_EXPORT_ROWS | 10000 | Excel 다운로드 최대 건수 |
| TIMEZONE | Asia/Seoul | 시간대 설정 |

설정을 변경하려면:
1. 변경할 설정의 **값** 입력란을 수정합니다.
2. 해당 행의 **[저장]** 버튼을 클릭합니다.

### 8.2 이상행위 탐지 규칙

등록된 탐지 규칙 목록을 확인할 수 있습니다.

| 열 | 설명 |
|----|------|
| 코드 | 규칙 코드 (R01~R10) |
| 규칙명 | 규칙 이름 |
| 심각도 | HIGH / MEDIUM / LOW / INFO |
| 조건유형 | 탐지 조건 유형 |
| 임계값 | 탐지 기준 수치 (예: 1000건) |
| 시간범위 | 적용 시간 범위 (예: 60분, 22:00~06:00) |
| 활성 | 활성/비활성 상태 |

### 8.3 해시 무결성 검증

접속기록이 위/변조되지 않았는지 SHA-256 해시 체인으로 검증합니다.

#### 검증 방법

1. **[검증 실행]** 버튼을 클릭합니다.
2. 당일 접속기록의 해시 체인을 순차적으로 검증합니다.
3. 검증 결과가 아래와 같이 표시됩니다:

#### 검증 결과

| 결과 | 의미 |
|------|------|
| **VALID** (초록) | 모든 접속기록의 해시 체인이 정상 |
| **INVALID** (빨강) | 위/변조가 의심되는 기록 발견 |

표시 정보:
- **검증 대상**: 검증한 총 기록 건수
- **정상**: 해시 체인이 일치하는 건수
- **위반**: 해시 체인이 불일치하는 건수

> **해시 체인이란?** 각 접속기록의 해시값을 계산할 때 이전 기록의 해시값을 포함합니다.
> 이를 통해 중간에 기록이 삭제되거나 수정된 경우 즉시 탐지할 수 있습니다.

## 9. FAQ

### Q. 접속기록관리 메뉴가 보이지 않습니다.
**A.** IT, SEC, ADMIN 역할이 부여된 계정만 접속 가능합니다. 관리자에게 권한을 요청하세요.

### Q. 수집을 시작했는데 기록이 수집되지 않습니다.
**A.** 아래 사항을 확인하세요:
1. 대상 DB의 Audit 기능이 활성화되어 있는지 확인
2. 수집 대상의 호스트/포트/DB정보가 올바른지 확인
3. DLM에서 대상 DB로 네트워크 연결이 가능한지 확인
4. 수집 대상의 상태가 ERROR인 경우 로그 확인

### Q. 해시 검증 결과가 INVALID입니다.
**A.** 접속기록이 DB에서 직접 수정/삭제된 경우 발생합니다. 보안 담당자에게 즉시 보고하고 원인을 조사하세요.

### Q. Excel 다운로드 시 사유 입력이 필요한 이유는?
**A.** 개인정보보호법에 따라 접속기록 다운로드는 감사 대상입니다. 모든 다운로드는 일시, 사용자, 사유와 함께 기록됩니다.

### Q. 이상행위 알림이 너무 많이 발생합니다.
**A.** 환경설정 > 이상행위 탐지 규칙에서 임계값을 조정하거나, 업무 특성에 맞지 않는 규칙을 비활성화할 수 있습니다.

### Q. DLM 내부 작업도 접속기록에 남나요?
**A.** 네, AOP Interceptor가 DLM 내부의 컨트롤러 호출을 자동으로 기록합니다 (수집방식: DLM_SELF).
