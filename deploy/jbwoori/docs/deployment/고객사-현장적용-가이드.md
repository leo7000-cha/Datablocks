## 1. 전체 흐름 한눈에 보기

```
개발 PC                    고객사
========                   ======

[소스코드]                  [WAS 서버] ←── Docker 설치
    |                          |
    | tar로 압축               | 압축 풀기
    | USB/SCP로 전송           |
    v                          v
[배포파일.tar.gz]  ------->  /app/Datablocks/
                               |
                               | .env.customer 에서
                               | DB IP만 수정
                               |
                               v
                            docker compose up -d --build
                               |
                               v
                            +-------+    +-----------+
                            |  DLM  |    | Privacy-AI|
                            +---+---+    +-----+-----+
                                |              |
                                +--------------+
                                |
                           네트워크 통신
                                |
                                v
                      [고객사 DB 서버]
                        MariaDB
                        192.168.x.x:3306
```

### 요약: 5줄로 정리

```
1. 개발PC에서 소스코드를 tar.gz로 압축
2. 고객사 WAS 서버에 전송
3. 고객사 WAS 서버에 Docker 설치 (한 번만)
4. .env.customer 파일에서 DB서버 IP 수정
5. docker compose up  →  끝!
```

## 3. STEP 1 — 개발PC에서 배포 파일 준비

### 어떤 파일을 가져가야 하나?

```
/app/Datablocks/              ← 이 폴더 전체를 가져감
│
├── DLM/                      ← Spring Boot 소스 (필수)
│   ├── Dockerfile
│   ├── src/
│   ├── build.gradle
│   ├── settings.gradle
│   ├── libs/
│   └── ...
│
├── DLM-Privacy-AI/           ← AI 서비스 소스 (필수)
│   ├── Dockerfile
│   ├── app/
│   ├── requirements.txt
│   └── ...
│
├── docker-compose.customer.yml  ← 고객사용 Docker 설정 (필수)
├── .env.customer                ← 고객사용 환경변수 (필수)
│
├── mariadb/                  ← DB 초기화 파일 (필수 - DB 설정용)
│   ├── init/
│   │   ├── cotdl_dump.sql.data    ← DB 생성 + 테이블 + 데이터 (전체)
│   │   └── cotdl_users.sql.data   ← DB 접속 유저 계정 생성 (별도 실행 필요)
│   └── conf.d/
│       └── custom-prod.cnf        ← MariaDB 운영 설정 (DBA에게 전달)
│
├── docs/                     ← 문서 (참고용)
│
│  ---- 아래는 가져가지 않아도 됨 ----
├── docker-compose.yml        ← 개발용 (가져가도 사용 안 함)
├── .env                      ← 개발용 (가져가도 사용 안 함)
└── .git/                     ← Git 히스토리 (불필요)
```

### (선택) DB 덤프 최신화

배포 전에 현재 개발 DB의 최신 데이터를 덤프 파일에 반영하려면:

```bash
# 개발PC에서 실행 — 최신 DB 데이터로 덤프 파일 갱신
docker exec dlm-mariadb mariadb-dump -u root -p'!Dlm1234' \
  --databases cotdl --single-transaction 2>/dev/null \
  > /app/Datablocks/mariadb/init/cotdl_dump.sql.data
```

> 이 파일에 **DB 생성 + 테이블 구조 + 데이터 전부** 포함됩니다.
> 유저 계정은 별도 파일(`cotdl_users.sql.data`)에 있으므로 따로 덤프 불필요.

### 압축하기

개발PC 터미널에서 실행:

```bash
# 개발PC에서 실행
cd /app

# .git 제외하고 압축 (용량 절약)
tar czf DLM-배포패키지.tar.gz \
    --exclude='Datablocks/.git' \
    --exclude='Datablocks/DLM/.gradle' \
    --exclude='Datablocks/DLM/build' \
    --exclude='Datablocks/DLM/dlm-agent/build' \
    --exclude='Datablocks/DLM-Privacy-AI/__pycache__' \
    --exclude='Datablocks/DLM-Privacy-AI/.venv' \
    Datablocks/

# 파일 크기 확인
ls -lh DLM-배포패키지.tar.gz
```

> 예상 크기: 약 200~400MB

## 4. STEP 2 — 고객사 서버에 파일 전송

### 방법 A: SCP (네트워크 전송)

```bash
# 개발PC에서 실행
# scp  [보낼파일]  [고객사계정]@[고객사IP]:[경로]

scp /app/DLM-배포패키지.tar.gz  root@192.168.1.50:/app/
```

### 방법 B: USB

```
1. USB에 DLM-배포패키지.tar.gz 복사
2. 고객사 서버에 USB 연결
3. USB 마운트 후 파일 복사:

   mount /dev/sdb1 /mnt/usb
   cp /mnt/usb/DLM-배포패키지.tar.gz /app/
   umount /mnt/usb
```

### 압축 풀기 (고객사 서버에서)

```bash
# 고객사 서버에서 실행
cd /app
tar xzf DLM-배포패키지.tar.gz

# 확인
ls /app/Datablocks/
# 결과: DLM/  DLM-Privacy-AI/  docker-compose.customer.yml  .env.customer  mariadb/  docs/
```

## 5. STEP 3 — 고객사 서버에 Docker 설치

> Docker가 이미 설치되어 있다면 이 단계를 건너뛰세요.
>
> 확인 방법: `docker --version` 입력 → 버전이 나오면 설치됨

### CentOS 7/8 / RHEL 8 (인터넷 가능)

```bash
# 1. 기존 Docker 제거 (혹시 옛날 버전이 있으면)
sudo yum remove -y docker docker-client docker-common docker-engine

# 2. 필요한 도구 설치
sudo yum install -y yum-utils

# 3. Docker 저장소 추가
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo

# 4. Docker 설치
sudo yum install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 5. Docker 시작 & 자동시작 등록
sudo systemctl start docker
sudo systemctl enable docker

# 6. 설치 확인
docker --version
docker compose version
```

### Rocky Linux 9 / RHEL 9 / AlmaLinux 9 (인터넷 가능)

> Rocky 9는 CentOS 8의 후속입니다. `dnf` 명령을 사용합니다.

```bash
# 1. 기존 Docker 제거 (혹시 옛날 버전이 있으면)
sudo dnf remove -y docker docker-client docker-common docker-engine podman buildah

# 2. 필요한 도구 설치
sudo dnf install -y dnf-plugins-core

# 3. Docker 저장소 추가 (Rocky 9는 centos 저장소를 사용합니다)
sudo dnf config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo

# 4. Docker 설치
sudo dnf install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 5. Docker 시작 & 자동시작 등록
sudo systemctl start docker
sudo systemctl enable docker

# 6. 설치 확인
docker --version
docker compose version
```

> **Rocky 9 주의사항**:
> - Rocky 9에는 `podman`이 기본 설치되어 있을 수 있습니다. 충돌 방지를 위해 STEP 1에서 제거합니다.
> - `yum` 대신 `dnf`를 사용합니다 (`yum`도 동작하지만 `dnf`가 공식 권장).
> - 방화벽이 `firewalld`로 활성화되어 있으면 포트 오픈이 필요합니다:
>   ```bash
>   sudo firewall-cmd --permanent --add-port=8080/tcp   # DLM 웹
>   sudo firewall-cmd --permanent --add-port=8000/tcp   # Privacy-AI
>   sudo firewall-cmd --reload
>   ```

### Ubuntu (인터넷 가능)

```bash
# 1. 기존 Docker 제거
sudo apt-get remove -y docker docker-engine docker.io containerd runc

# 2. 필요한 도구 설치
sudo apt-get update
sudo apt-get install -y ca-certificates curl gnupg

# 3. Docker 공식 GPG 키 추가
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

# 4. Docker 저장소 추가
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/ubuntu \
  $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 5. Docker 설치
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 6. Docker 시작 & 자동시작 등록
sudo systemctl start docker
sudo systemctl enable docker

# 7. 설치 확인
docker --version
docker compose version
```

### 폐쇄망 (인터넷 안 되는 서버)

```
인터넷이 되는 PC에서 미리 다운로드:

1. 아래 URL에서 서버 OS에 맞는 RPM/DEB 파일 다운로드:

   CentOS 7:   https://download.docker.com/linux/centos/7/x86_64/stable/Packages/
   CentOS 8:   https://download.docker.com/linux/centos/8/x86_64/stable/Packages/
   Rocky 9:    https://download.docker.com/linux/centos/9/x86_64/stable/Packages/
   Ubuntu:     https://download.docker.com/linux/ubuntu/dists/

2. 4개 파일을 다운로드:
   - docker-ce-*.rpm (또는 .deb)
   - docker-ce-cli-*.rpm
   - containerd.io-*.rpm
   - docker-compose-plugin-*.rpm

3. USB로 고객사 서버에 복사

4. 고객사 서버에서:

   # CentOS 7/8
   sudo yum install -y ./docker-ce-*.rpm ./docker-ce-cli-*.rpm \
                        ./containerd.io-*.rpm ./docker-compose-plugin-*.rpm

   # Rocky 9 / RHEL 9 / AlmaLinux 9
   sudo dnf install -y ./docker-ce-*.rpm ./docker-ce-cli-*.rpm \
                        ./containerd.io-*.rpm ./docker-compose-plugin-*.rpm

   # 공통 - 시작 & 자동시작
   sudo systemctl start docker
   sudo systemctl enable docker
```

## 6. STEP 4 — MariaDB 설정 (고객사 DB 서버)

> MariaDB는 고객사가 이미 설치해 둔 것을 사용합니다.
> 우리가 할 일 3가지:
>
> | 순서 | 파일 | 내용 |
> |------|------|------|
> | 4-1 | `custom-prod.cnf` (DBA 전달) | MariaDB 필수 설정 확인 |
> | 4-2 | `cotdl_dump.sql.data` | DB 생성 + 테이블 + 데이터 (한 번에 복원) |
> | 4-3 | `cotdl_users.sql.data` | DB 접속 유저 계정 생성 **(별도 실행)** |

### 4-1. MariaDB 필수 설정 확인

DLM이 정상 동작하려면 MariaDB에 아래 설정이 **반드시** 필요합니다:

```
┌──────────────────────────────────────────────────────────┐
│  MariaDB 필수 설정 (my.cnf 또는 고객사 설정파일)            │
│                                                           │
│  [mysqld]                                                 │
│  character-set-server = utf8mb4          ← 한글 지원       │
│  collation-server = utf8mb4_general_ci   ← 정렬 규칙       │
│  lower_case_table_names = 1              ← 대소문자 무시 ★  │
│  sql_mode = "ANSI_QUOTES,..."            ← DLM 쿼리 호환 ★ │
│  default_time_zone = 'Asia/Seoul'        ← 시간대           │
│  max_connections = 500                   ← 연결 수          │
│  max_allowed_packet = 64M                ← 대용량 쿼리      │
│                                                           │
│  ★ = 이것이 없으면 DLM이 오류 발생                          │
└──────────────────────────────────────────────────────────┘
```

**고객사 DBA에게 전달할 설정 파일**: `mariadb/conf.d/custom-prod.cnf`

```bash
# 고객사 DB 서버에서 현재 설정 확인
mysql -u root -p -e "SHOW VARIABLES LIKE 'lower_case_table_names';"
mysql -u root -p -e "SHOW VARIABLES LIKE 'sql_mode';"
mysql -u root -p -e "SHOW VARIABLES LIKE 'character_set_server';"
```

> **중요**: `lower_case_table_names`과 `sql_mode`에 `ANSI_QUOTES`가 없으면
> DLM에서 SQL 오류가 발생합니다. DBA에게 설정 변경을 요청하세요.

### 4-2. 데이터베이스 & 테이블 생성

```bash
# 고객사 DB 서버에서 실행 (또는 원격 접속)

# 1. DLM DB 덤프 파일을 DB 서버로 복사
scp /app/Datablocks/mariadb/init/cotdl_dump.sql.data  root@DB서버IP:/tmp/

# 2. DB 서버에서 실행
mysql -u root -p < /tmp/cotdl_dump.sql.data

# 3. 확인
mysql -u root -p -e "SHOW DATABASES;"
# cotdl 이 보이면 성공

mysql -u root -p cotdl -e "SHOW TABLES;"
# 테이블 목록이 나오면 성공
```

### 4-3. DB 계정 생성

```bash
# 1. 계정 생성 SQL을 DB 서버로 복사
scp /app/Datablocks/mariadb/init/cotdl_users.sql.data  root@DB서버IP:/tmp/

# 2. DB 서버에서 실행
mysql -u root -p < /tmp/cotdl_users.sql.data

# 3. 확인
mysql -u root -p -e "SELECT User, Host FROM mysql.user WHERE User = 'cotdl';"
# cotdl 계정이 보이면 성공
```

> **참고**: 고객사에서 별도 DB 계정명/비밀번호를 요구하면,
> `cotdl_users.sql.data`를 수정하거나 DBA에게 계정 생성을 요청하세요.
> 이 경우 `.env.customer`의 DB 비밀번호도 함께 변경해야 합니다.

### 4-4. WAS 서버 → DB 서버 통신 확인

```bash
# WAS 서버(DLM 서버)에서 실행
# DB 서버의 3306 포트에 접속 가능한지 확인

# 방법 1: telnet
telnet 192.168.1.100 3306
# Connected 라고 나오면 성공. Ctrl+] 후 quit 으로 종료

# 방법 2: nc (netcat)
nc -zv 192.168.1.100 3306
# succeeded! 가 나오면 성공

# 방법 3: mysql 클라이언트가 설치되어 있다면
mysql -h 192.168.1.100 -u cotdl -p cotdl -e "SELECT 1;"
```

> 접속이 안 되면? → 고객사 네트워크/방화벽 담당자에게 **WAS서버IP → DB서버IP:3306** 방화벽 오픈 요청

### 7-3. DB 연결 확인

로그인 후 대시보드가 정상적으로 표시되면 DB 연결 성공입니다.

에러 화면이 나오면:

```bash
# DLM 로그에서 DB 연결 에러 확인
docker logs dlm-app 2>&1 | grep -i "error\|exception\|refused\|timeout"
```

### 흔한 문제와 해결

| 증상 | 원인 | 해결 |
|------|------|------|
| 웹 접속 안 됨 | 방화벽 | WAS 서버의 8080 포트 방화벽 오픈 |
| 로그인 후 에러 | DB 연결 실패 | `.env.customer`의 DB IP 확인 |
| `Connection refused` | DB 서버 방화벽 | WAS→DB 3306 포트 방화벽 오픈 |
| `Access denied` | DB 계정 문제 | cotdl 계정이 WAS서버 IP에서 접속 허용되는지 확인 |
| `Unknown database 'cotdl'` | DB 미생성 | STEP 4-2 다시 실행 |
| `ANSI_QUOTES` 관련 에러 | sql_mode 미설정 | STEP 4-1 MariaDB 설정 확인 |
| 컨테이너 재시작 반복 | 메모리 부족 | `docker logs dlm-app`으로 확인, 서버 메모리 확인 |

### DB 연결 에러 상세 해결

```bash
# 1. WAS 서버에서 DB 서버로 직접 접속 테스트
#    (Docker 안에서 테스트)
docker exec -it dlm-app sh -c "wget -qO- --timeout=3 DB서버IP:3306 || echo '접속 실패'"

# 2. 접속 실패 → 방화벽 문제
#    고객사 네트워크 담당자에게 요청:
#    "WAS서버(10.20.30.50) → DB서버(10.20.30.40):3306 TCP 방화벽 오픈해주세요"

# 3. 접속은 되는데 Access denied → 계정 권한 문제
#    DB 서버에서 실행:
mysql -u root -p -e "GRANT ALL PRIVILEGES ON cotdl.* TO 'cotdl'@'WAS서버IP' IDENTIFIED BY '!Dlm1234'; FLUSH PRIVILEGES;"
```

## 10. STEP 8 — 운영 관리

### 자주 쓰는 명령어

```bash
# ★ 모든 명령은 /app/Datablocks/ 에서 실행

cd /app/Datablocks

# ---------- 상태 확인 ----------
docker ps                          # 실행 중인 컨테이너 확인
docker logs -f dlm-app             # DLM 실시간 로그 (Ctrl+C로 종료)
docker logs -f dlm-privacy-ai     # AI 실시간 로그

# ---------- 중지 / 시작 ----------
# 전체 중지
docker compose -f docker-compose.customer.yml --env-file .env.customer down

# 전체 시작 (이미 빌드된 상태)
docker compose -f docker-compose.customer.yml --env-file .env.customer up -d

# DLM만 재시작
docker compose -f docker-compose.customer.yml --env-file .env.customer restart dlm

# ---------- 업데이트 (소스 변경 후) ----------
# DLM만 재빌드 & 재시작
docker compose -f docker-compose.customer.yml --env-file .env.customer up -d --build dlm

# AI만 재빌드 & 재시작
docker compose -f docker-compose.customer.yml --env-file .env.customer up -d --build dlm-privacy-ai

# 전체 재빌드
docker compose -f docker-compose.customer.yml --env-file .env.customer up -d --build
```

### 명령어가 너무 길다면 — 단축 명령 설정

```bash
# ~/.bashrc에 추가 (한 번만 실행)
cat >> ~/.bashrc << 'EOF'

# DLM 단축 명령어
alias dlm-start='cd /app/Datablocks && docker compose -f docker-compose.customer.yml --env-file .env.customer up -d'
alias dlm-stop='cd /app/Datablocks && docker compose -f docker-compose.customer.yml --env-file .env.customer down'
alias dlm-build='cd /app/Datablocks && docker compose -f docker-compose.customer.yml --env-file .env.customer up -d --build'
alias dlm-log='docker logs -f dlm-app'
alias dlm-ai-log='docker logs -f dlm-privacy-ai'
alias dlm-ps='docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"'
EOF

# 적용
source ~/.bashrc

# 이제 이렇게 쓸 수 있습니다:
dlm-start     # 시작
dlm-stop      # 중지
dlm-build     # 재빌드
dlm-log       # 로그 보기
dlm-ps        # 상태 확인
```

### 서버 재부팅 시 자동 시작

Docker에 `restart: unless-stopped` 설정이 되어 있으므로,
서버가 재부팅되면 **Docker만 자동 시작되면** 컨테이너도 자동 시작됩니다.

```bash
# Docker 자동 시작 확인 (이미 STEP 3에서 설정함)
sudo systemctl is-enabled docker
# 결과: enabled 이면 OK
```

### 로그 파일 위치

```bash
# DLM 애플리케이션 로그 (Docker 볼륨 안)
docker exec dlm-app ls /app/logs/

# 호스트에서 직접 보기
docker volume inspect dlm-app-logs
# "Mountpoint" 값이 실제 경로
# 예: /var/lib/docker/volumes/dlm-app-logs/_data/
```

### 소스 업데이트 (패치 적용)

